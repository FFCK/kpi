# Nginx Static App Deployment (App2 & App3)

**Date**: 2025-12-21
**Status**: ✅ Implemented
**Components**: Nginx, Docker Compose, Traefik, Nuxt SSG

## Overview

This document describes the infrastructure for serving Nuxt applications (app2 and app3) as static files via Nginx, with environment-specific builds and CORS configuration.

## Architecture

### Components

```
┌─────────────────────────────────────────────────────────────────┐
│                         Traefik (Reverse Proxy)                  │
│  - app.kpi.localhost → nginx_app2 (dev)                         │
│  - app.kayak-polo.info → nginx_app2 (prod)                      │
│  - app3.localhost → node3 (dev server)                          │
└─────────────────────────────────────────────────────────────────┘
                              │
            ┌─────────────────┴───────────────────┐
            │                                     │
     ┌──────▼──────┐                     ┌────────▼────────┐
     │ Nginx       │                     │ PHP/Apache      │
     │ Alpine      │                     │ (kpi container) │
     └─────────────┘                     └─────────────────┘
            │                                     │
     ┌──────▼──────────────────┐         ┌───────▼────────┐
     │ app2/.output/public/    │         │ API Endpoints  │
     │ - index.html            │         │ /api/*         │
     │ - _nuxt/...             │         │ /api2/*        │
     │ - assets/...            │         │ custom files   │
     └─────────────────────────┘         └────────────────┘
```

### Nginx Container

**Service name**: `nginx_app2`
**Image**: `nginx:alpine`
**Purpose**: Serve static files generated by Nuxt SSG
**Volumes**:
- `sources/app2/.output/public:/usr/share/nginx/html:ro` (read-only)
- `docker/config/nginx-app2.conf:/etc/nginx/conf.d/default.conf:ro`
- `${HOST_NGINX_LOG_PATH}:/var/log/nginx/`

**Labels** (Traefik):
- Dev: `app.${KPI_DOMAIN_NAME}` (e.g., `app.kpi.localhost`)
- Prod: `app.${KPI_DOMAIN_NAME}` (e.g., `app.kayak-polo.info`)

## Nginx Configuration

**File**: `docker/config/nginx-app2.conf`

```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # SPA routing: fallback to index.html for all routes
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets for 1 year
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_types text/plain text/css text/xml text/javascript
               application/x-javascript application/xml+rss
               application/json application/javascript;
}
```

## Build & Deployment Workflow

### Development Environment

#### Option 1: Dev Server (Hot Reload)
```bash
# Start Node.js dev server
make docker_dev_up
make app2_dev  # Accessible at https://kpi-node.localhost
```

#### Option 2: Static Build (Test Nginx)
```bash
# Generate static files with dev config
make app2_generate_dev

# Restart nginx to serve new files
docker restart kpi_nginx_app2

# Access at https://app.kpi.localhost
```

### Production Environment

#### Build Process (No Permanent Node.js Container Required)

```bash
# On production server
cd /path/to/kpi
git pull origin main

# Generate static files using temporary container
make app2_generate_prod

# Restart nginx
make docker_prod_restart
```

**Technical Details**:
- `make app2_generate_prod` uses `docker run --rm` with temporary Node.js container
- No permanent Node.js container needed in production
- Uses `.env.production` for environment variables

#### Makefile Commands

**Development** (requires permanent Node container):
```makefile
app2_generate_dev:
    $(DOCKER_EXEC_NODE_NON_INTERACTIVE) sh -c "npx dotenv-cli -e .env.development -- nuxt generate"
```

**Production** (uses temporary container):
```makefile
app2_generate_prod:
    @echo "🔨 Building app2 for production using temporary Node.js container..."
    docker run --rm -v "$(CURDIR)/sources/app2:/app" -w /app node:20-alpine sh -c "npm ci && npx dotenv-cli -e .env.production -- nuxt generate"
    @echo "✅ Build complete! Files are in sources/app2/.output/public/"
```

```typescript
const baseUrl = process.env.BASE_URL ?? ''
```

## CORS Configuration

### Global PHP Auto-Prepend

All PHP files (including API, custom files, api2) automatically include CORS headers via PHP auto-prepend.

**Files**:
- `docker/config/auto-prepend-cors.php` - CORS logic
- `docker/config/php-auto-prepend.ini` - PHP configuration

**Allowed Origins**:
```php
// Production
$origin === "https://kayak-polo.info"
$origin === "https://www.kayak-polo.info"
$origin === "https://app2.kayak-polo.info"

// Development
$origin === "http://localhost:9000"
$origin === "http://localhost:9001"
$origin === "http://localhost:9002"
$origin === "http://localhost:3002"
$origin === "https://kpi-node.localhost"
$origin === "https://app.kpi.localhost"  // Nginx static app

// Pattern match for any .localhost domain
preg_match('/^https?:\/\/.*\.localhost$/', $origin)
```

**Headers Set**:
- `Access-Control-Allow-Origin`: Specific origin (not wildcard)
- `Access-Control-Allow-Credentials`: true
- `Access-Control-Allow-Methods`: GET, PUT, POST, DELETE, OPTIONS, PATCH
- `Access-Control-Allow-Headers`: Origin, Content-Type, X-Auth-Token, X-Requested-With, Authorization, Cache-Control, Pragma, Expires
- `Access-Control-Max-Age`: 1000

### Apache Configuration

**File**: `docker/config/000-default.conf`

```apache
<Directory /var/www/html>
    # CORS headers are managed dynamically by PHP (see auto-prepend-cors.php)
    # Removed static CORS headers to avoid conflicts with PHP-managed headers
</Directory>
```

**Key Change**: Removed all static `Header always set Access-Control-*` directives from Apache to avoid duplicate headers.

### Dev vs Preprod vs Prod Differences

| Aspect | Development | Pre-production | Production |
|--------|-------------|----------------|------------|
| Apache Config | Volume mount | Volume mount | COPY in Dockerfile |
| CORS Auto-Prepend | COPY in Dockerfile (all envs) | COPY in Dockerfile | COPY in Dockerfile |
| Nginx Config | `nginx-app2.conf` (dev) | `nginx-app2-prod.conf` | `nginx-app2-prod.conf` |
| Nginx Cache | Disabled (`sendfile off`) | Enabled (`sendfile on`) | Enabled (`sendfile on`) |
| Build Command | `make app2_generate_dev` | `make app2_generate_prod` | `make app2_generate_prod` |
| Node Container | Permanent (for hot-reload) | Temporary (docker run --rm) | Temporary (docker run --rm) |
| Domain | `app.kpi.localhost` | `app.{PREPROD_DOMAIN}` | `app.kayak-polo.info` |
| Auto-restart after build | Yes (Makefile) | Manual | Manual |

## Docker Compose Configuration

### Development (compose.dev.yaml)

```yaml
nginx_app2:
    container_name: ${APPLICATION_NAME}_nginx_app2
    image: nginx:alpine
    volumes:
        - ../sources/app2/.output/public:/usr/share/nginx/html:ro
        - ./config/nginx-app2.conf:/etc/nginx/conf.d/default.conf:ro
        - ${HOST_NGINX_LOG_PATH}:/var/log/nginx/
    networks:
        - network_kpi
        - traefiknetwork
    labels:
        - "traefik.enable=true"
        - "traefik.http.routers.nginx_app2.rule=Host(`app.${KPI_DOMAIN_NAME}`)"
        - "traefik.http.routers.nginx_app2.entrypoints=websecure"
        - "traefik.http.routers.nginx_app2.tls=true"
```

### Pre-production (compose.preprod.yaml)

```yaml
nginx_app2:
    container_name: ${APPLICATION_NAME}_nginx_app2
    image: nginx:alpine
    volumes:
        - ../sources/app2/.output/public:/usr/share/nginx/html:ro
        - ./config/nginx-app2-prod.conf:/etc/nginx/conf.d/default.conf:ro
        - ${HOST_NGINX_LOG_PATH}:/var/log/nginx/
    labels:
        - "traefik.http.routers.nginx_app2_preprod.rule=Host(`app.${KPI_DOMAIN_NAME}`)"
        - "traefik.http.routers.nginx_app2_preprod.tls.certresolver=myresolver"
        - "traefik.http.services.nginx_app2_preprod.loadbalancer.server.port=80"
```

**Note**: Pre-production uses production nginx config (`nginx-app2-prod.conf`) with optimized caching.

### Production (compose.prod.yaml)

```yaml
nginx_app2:
    # Same as preprod
    volumes:
        - ../sources/app2/.output/public:/usr/share/nginx/html:ro
        - ./config/nginx-app2-prod.conf:/etc/nginx/conf.d/default.conf:ro
    labels:
        - "traefik.http.routers.nginx_app2.tls.certresolver=myresolver"
```

**Note**: Production does NOT mount `000-default.conf` as volume for Apache - it's baked into the Docker image via Dockerfile COPY.

## Dockerfile Configuration

### Development (Dockerfile.dev.web)

```dockerfile
# Copy PHP auto-prepend configuration for CORS
COPY php-auto-prepend.ini /usr/local/etc/php/conf.d/
COPY auto-prepend-cors.php /var/www/html/commun/auto-prepend-cors.php
RUN chmod 644 /var/www/html/commun/auto-prepend-cors.php

# Copy Apache vhost configuration with CORS and .htaccess support
COPY 000-default.conf /etc/apache2/sites-enabled/000-default.conf
RUN chmod 644 /etc/apache2/sites-enabled/000-default.conf
```

**Note**: Volume mount in `compose.dev.yaml` overrides the COPY for `000-default.conf`, allowing modification without rebuild.

### Production (Dockerfile.prod.web)

Same COPY commands, but no volume mount override - config is immutable.

## Troubleshooting

### Issue: 403 Forbidden

**Cause**: No `index.html` in `.output/public/` (SSR mode enabled)
**Solution**: Set `ssr: false` in `nuxt.config.ts` and regenerate

### Issue: Wrong API URL

**Cause**: Cached build with wrong environment
**Solution**:
```bash
# Force rebuild with correct env
rm -rf sources/app2/.output
make app2_generate_dev  # or app2_generate_prod
```

### Issue: Service Worker Errors

**Cause**: Old Service Worker from previous build
**Solution**: Temporarily disable PWA in `nuxt.config.ts`:
```typescript
pwa: {
  disable: true
}
```

### Issue: Duplicate CORS Headers

**Cause**: Both Apache and PHP setting headers
**Solution**:
1. Remove all `Header always set Access-Control-*` from Apache config
2. Ensure `auto-prepend-cors.php` is loaded via `php-auto-prepend.ini`
3. Rebuild Docker image: `make docker_dev_rebuild`

### Issue: Build Fails in Production

**Cause**: Missing Node.js dependencies
**Solution**: Use `npm ci` instead of `npm install` in temporary container (already implemented in `make app2_generate_prod`)

## Performance Considerations

### Nginx Caching
- Static assets cached for 1 year (`expires 1y`)
- Cache-Control header: `public, immutable`
- Reduces bandwidth and improves load times

### Gzip Compression
- Enabled for text/CSS/JS/JSON
- Reduces transfer size by ~70%

### Read-Only Volumes
- `.output/public` mounted as `:ro` for security
- Prevents accidental modification

## Security

### Headers
- `X-Frame-Options: SAMEORIGIN` - Prevents clickjacking
- `X-Content-Type-Options: nosniff` - Prevents MIME sniffing
- `X-XSS-Protection: 1; mode=block` - XSS protection

### CORS
- Explicit origin matching (no wildcards in production)
- Credentials allowed only for authorized origins
- Preflight OPTIONS requests handled correctly
